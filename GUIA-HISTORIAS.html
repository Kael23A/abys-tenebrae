Guía del archivo

Cómo añadir nuevas historias sin romper nada

El único archivo que necesitas editar

Toda la web funciona a partir de un solo archivo central:

rth-historias.js

Cuando añades una historia ahí, automáticamente aparece en la lista de su colección, en las historias recientes de la portada, y el contador se actualiza solo. No tienes que tocar ningún otro archivo.

✓ REGLA DE ORO: Cada historia nueva = un bloque nuevo en rth-historias.js + su archivo HTML.

Paso a paso

Paso 01 — Escribe o recibe la historia en HTML

Cada historia tiene su propio archivo .html. Usa como plantilla el archivo la-voz-del-numero-equivocado.html (Oscuridad) o el-ultimo-turno.html (Anomalía).

El nombre del archivo debe ser: palabras en minúsculas separadas por guiones. Sin espacios, sin acentos. Ejemplo: el-susurro-del-rio.html

Paso 02 — Abre rth-historias.js

Busca esta línea al inicio del array:

/* ══ AÑADE AQUÍ ARRIBA LAS NUEVAS ══ */

Pega tu bloque justo después de esa línea, antes de los bloques existentes. Las más recientes van primero.

Paso 03 — Copia este bloque y rellena los campos 
/* ══ NUEVA HISTORIA ══ */
{
  id:        "nombre-del-archivo",        // sin .html
  titulo:    "Título completo",
  gancho:    "Frase que engancha al lector...",
  coleccion: "oscuridad",                  // oscuridad | anomalia | mitos
  genero:    "creepypasta",               // ver tabla abajo
  minutos:   7,                            // número sin comillas
  etiquetas: ["Creepypasta", "Paranormal"]  // máximo 3
},
⚠ IMPORTANTE: La coma al final del bloque }, es obligatoria si hay más historias después. El último bloque del array NO lleva coma.
Paso 04 — Historia aún sin página terminada

Si quieres que aparezca como "Próximamente" sin ser clickeable, añade este campo extra:

proximo: true

Cuando la página esté lista, simplemente borra esa línea. Se activa sola.

Paso 05 — Sube los archivos a Netlify

Siempre sube juntos: el .html de la historia nueva + el rth-historias.js actualizado. Nada más. El resto de la web no necesita cambios.

Estructura del HTML de una historia

Dentro de hay cuatro clases especiales para formatear el texto:

.dialogo

Para líneas de diálogo de personajes humanos. Aparece en cursiva con borde izquierdo.

<p class="dialogo">—Aquí va lo que dice el personaje.</p>

.aria / .tension

Para voces de IA o momentos de giro dramático. Más luminoso, destaca en el color de la colección.

<p class="tension">El teléfono había estado apagado toda la noche.</p>

.noche

Para marcas de tiempo o separadores entre secciones. Aparece en fuente monoespaciada discreta.

<span class="noche">// Segunda noche — 1:12 a.m.</span>

.remate

Para el párrafo final, el cierre de la historia. Aparece separado, en cursiva y tono más suave.

<p class="remate">Y nadie volvió a saber de él.</p>

<hr>

Una línea horizontal fina para separar secciones dentro de la historia.

<hr>

Checklist antes de subir

✓ El archivo .html de la historia tiene nombre sin espacios ni acentos
✓ El campo id en rth-historias.js coincide exactamente con el nombre del archivo
✓ El bloque nuevo va AL INICIO del array (antes de los existentes)
✓ Si hay más bloques después, el tuyo termina con }, (con coma)
✓ Subiste AMBOS archivos: el .html + rth-historias.js
✗ NO tocar: rth-shared.css, rth-neon.js, index.html, oscuridad.html, anomalia.html, mitos.html — a menos que sea para corregir algo específico. Un error ahí puede afectar toda la web.
